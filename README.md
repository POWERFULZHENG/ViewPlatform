#### 1. 仓库定位

ViewPlatform 是一个基于 **Qt 框架（C++ 开发）** 构建的桌面应用平台代码仓库，核心目标是开发具备多模块功能的桌面端应用程序。

#### 2. 核心技术栈

从文件类型和命名可明确核心技术体系：

- 开发框架：Qt（`.pro` 是 Qt 项目构建文件、`.ui` 为 Qt Designer 界面文件、`.ts` 是 Qt 国际化翻译文件）；
- 开发语言：C++（`.cpp/.h` 源文件为核心代码载体）；
- 数据库：MySQL（`mysqlhelper.cpp/h` 封装了 MySQL 数据库的操作逻辑）；
- 版本管理：Git（`.gitignore` 定义 Git 忽略规则）。

#### 3. 核心功能模块

仓库按功能拆分出多个核心模块，对应文件及能力如下：

|         模块相关文件          |                           功能说明                           |
| :---------------------------: | :----------------------------------------------------------: |
|      `loginwidget.cpp/h`      |             登录窗口模块，实现应用的登录鉴权功能             |
|    `mainwindow.*`（含.ui）    |    主窗口模块，作为整个应用的核心界面载体，整合各功能模块    |
|       `*dbhelper.cpp/h`       | 数据库操作层：- `basedbhelper`：基础数据库操作封装（通用逻辑）；- `logdbhelper`：日志专属数据库操作；- `mysqlhelper`：MySQL 数据库适配 |
|    `logtablewidget.cpp/h`     |         日志表格模块，负责日志数据的展示、查询、管理         |
|   `algorithmsdatawidget.*`    |       算法数据模块，处理算法相关数据的可视化、交互逻辑       |
|       `llmwidget.cpp/h`       | LLM（大语言模型）模块，集成大语言模型相关功能（如对话、推理等） |
|      `apiconfigdialog.*`      |         API 配置对话框，管理 API 地址、密钥等配置项          |
| `configwidget.*`/`config.ini` |        应用配置模块，处理界面配置、全局参数配置等逻辑        |

#### 4. 工程配套能力

- 国际化：`MyQtCreator_en_AS.ts` 支持英文（南非）的国际化翻译，适配多语言场景；
- 资源管理：`icon.qrc` 统一管理应用图标等资源文件；
- 合规性：包含 `LICENSE` 许可证文件，明确开源授权规则；
- 工程构建：`MyQtCreator.pro` 作为 Qt 工程构建入口，定义编译、链接规则。

#### 5. 整体总结

该仓库是一个面向桌面端的 **多功能 Qt 应用开发骨架**，集成了 MySQL 数据库操作、配置管理、日志管理、算法数据展示、LLM 能力对接等核心能力，可作为数据分析、日志监控、AI 辅助工具等桌面应用的开发基础。### ViewPlatform 仓库介绍
